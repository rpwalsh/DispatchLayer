# Production Demo Data

This folder stores raw data artifacts used to run realistic local demos.

## Why It Exists

- allow reproducible dashboard/API behavior without private plant access
- keep demo context grounded in captured source data

## Typical Contents

- site catalog files
- time-series source captures
- metadata files for selected assets/systems

## Important Limits

- this is demo-support data, not plant historian truth
- completeness varies by source and capture window
- API and UI should surface missing fields transparently
